The whole idea, one screenshot

Not "yes."
"Yes, because."

Ask a question and the answer arrives with its receipt: the exact facts and rules that produced it, traced like a math proof. If the answer can't be derived from what it knows, it says so — "not derivable" — instead of guessing. That single behavior is what makes it trustworthy where chatbots aren't.

The AI language model (optional, local or remote) only translates and suggests. A small, exact engine — facts, logic rules, SQLite underneath — does all of the answering.

Three guarantees

The rules of the house

I.

Every answer shows its work

Proofs down to the given facts, on every query — for the user, the auditor, and the appeal. The decision log keeps every answer on record.

II.

Nothing gets in without a test

AI-drafted rules must reproduce your examples and preserve your pinned decisions. AI-written code is inspected, then run in a locked cage. Fail → rejected, never saved.

III.

Local, small, and honest

Pure-stdlib Python on one machine; your data never leaves it. No model connected? It runs as a purely symbolic engine — rules, proofs, and the HTTP service all still work.

Real questions it answers

examples/access_audit.py

"Who can touch prod — and why?"

Roles, inheritance, permissions, one suspended contractor. Proof chains for the auditor, what-if analysis by switching a rule off, and a typo-proof schema. source →

examples/eligibility_screening.py

"Do I qualify — and why not?"

Benefit rules with age/income thresholds and "unless disqualified." Every verdict carries the derivation an appeal would demand; a pinned case blocks a sloppy rewrite. source →

examples/service_sidecar.py

"Can my app just ask over HTTP?"

Yes — provenas-serve runs the same knowledge base as a local decision service, proofs riding along in every JSON response. source →

All three run offline in seconds — no AI model, nothing extra to install — and the test suite runs them too, so they can't drift from the code.

How it works

One shape, applied everywhere

The AI proposes

Translates your English into a tiny structured action; drafts candidate rules; drafts small tools. Suggestions only — it is never the authority.

→
The fabric verifies

An exact rule engine (with "unless" and number checks) answers from facts + rules and produces the proof. Proposals are tested against examples, pinned cases, and a sandbox before admission.

→
SQLite remembers

Facts, rules (with on/off history), tools, pinned cases, the schema, and an append-only log of every change and every decision — one portable file.

The fine print, up front It only knows what you teach it (it complements search/RAG, doesn't replace it). English understanding tracks the model you connect — the gates make a weak model safe, not smart. One machine, one writer, comfortable into the hundreds of thousands of facts.
